Pipeline ETL para Risco de Crédito

Contexto:

Estudo de caso em Engenharia de Dados voltado à estruturação de dados brutos de risco de crédito em um banco de dados relacional Oracle. O projeto simula um pipeline de dados utilizado no setor financeiro, abrangendo modelagem relacional, ingestão de dados, processamento em lotes e validação pós-carga.

Abordagem:

Modelagem relacional, criação de DDLs no Oracle, definição de chaves primárias e estrangeiras, ingestão de arquivos CSV com Python e Pandas, processamento em chunks para grandes volumes de dados, tratamento de valores nulos, resolução de erros durante a carga e consultas de validação para garantir qualidade e consistência dos dados..

Python · Oracle · SQL · Pandas · Data Engineering · ETL · Processamento em Lotes · Modelagem Relacional · Qualidade de Dados
